# war ディレクトリ非推奨化・削除計画

## 目的
- ルート直下のフォルダ構成を標準化し、`war/` の扱いを明確化する。
- いきなり削除せず、判定条件を満たした場合に次フェーズで削除できる状態を作る。

## 現状
- 調査日時: 2026-02-15
- 軽量調査（`rg`）結果:
  - `war/` 参照は `.gitignore` のみ
  - `README.md` / `AGENTS.md` / `src` / `deploy_qmaclone_tomcat9.ps1` / `pom.xml` に明示参照なし
- 現時点では `war/` は「常用の実行経路」ではなく、過去運用由来の残存ディレクトリである可能性が高い。

## 方針
- `war/` は新規運用で使用しない「非推奨」扱いとする。
- 本ドキュメントの削除ゲートを満たすまで、実体削除は実施しない。

## 削除ゲート（中庸）
1. 参照ゼロ確認
- `war/` への参照がコード・スクリプト・ドキュメントから消えていること
- `.gitignore` の `war/` ルール整理方針が確定していること

2. 本番同等デプロイ成功
- 通常デプロイ手順で配備し、以下を確認
  - `/QMAClone-1.0-SNAPSHOT/` が HTTP 200
  - `/tv.dyndns.kishibe.qmaclone.QMAClone/service` が HTTP 405

## 次フェーズ手順（削除実行時）
1. `war/` 参照の再検索
2. `.gitignore` の `war/` 関連行を見直し
3. `war/` 実体削除
4. `mvn compile` とデプロイ確認（200/405）
5. 問題なければコミット

## ロールバック
- 削除実行フェーズで問題が出た場合は、Git で `war/` を復元する。
- 復元後、参照箇所を再調査して削除ゲート未達として扱う。

## 実行結果（2026-02-15）
- `war/` 実体を削除し、`war/WEB-INF/web.xml` を削除した。
- `.gitignore` の `war/` 関連行を削除した。
- `mvn compile` は成功した。
- `war/` 実体は存在しないことを確認した。

### 実行コマンド
- `mvn compile`
- `rg -n "war/"`
- `rg -n "\\bwar\\b"`

### 補足
- `\\bwar\\b` は `pom.xml` の `<packaging>war</packaging>` や、WAR成果物を扱う配備スクリプト記述にも一致するため、文字列ゼロ化は適用対象を分離して扱う。
