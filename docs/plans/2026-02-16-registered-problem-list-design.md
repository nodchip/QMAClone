# 登録問題一覧 UI/UX 改善デザイン

## 背景
- 現在の「正解率統計」は名称と実態が一致していない。
- 実際の用途は「登録した問題の状態確認と管理」であり、グラフ中心の統計閲覧ではない。
- メインコンテンツは既存の共通パーツ（類似問題一覧 / 問題検索系）を活用しているため、共通性を保った改善が必要。

## 目的
- ユーザー目線で用途が即理解できる画面名・導線に改める。
- 管理判断に必要な情報へ集中し、不要なグラフ表示を除去する。
- 共通パーツを壊さず、画面設定で最適化する。

## 画面名称
- 変更前: 正解率統計
- 変更後: 登録問題一覧

## 上部説明文（固定3行）
- 登録した問題の正答率と状態を確認できます。
- 問題を登録・投稿した後は「情報更新」で最新状態を反映します。
- 一括消去は設定画面から実行できます。

## メイン表示方針
- グラフは表示しない。
- メインは共通テーブルを利用し、初期表示列を管理用途向けに固定する。

## 初期表示列
1. 問題番号
2. ジャンル
3. 形式
4. 問題文プレビュー
5. 正答率
6. 回答数
7. 指摘
8. 操作

## ソート仕様
- 既定ソート: 回答数降順 -> 正答率昇順
- ヘッダクリック: 単一列ソートへ切替、再クリックで昇降トグル
- 必要に応じて「既定ソートへ戻す」を上部に配置可能

## 実装方針
- 共通パーツ本体を大きく変更せず、この画面側の設定で完結させる。
- 既存の検索画面・類似問題一覧の挙動を維持する。
- 非表示列（作問者、画像/動画有無、良問等）は詳細表示側に残す。

## 受け入れ条件
- 画面名から用途が理解できる。
- 上部3行説明で操作導線（更新、ソート、一括消去）が明確。
- 初期表示列が8列で表示される。
- 初期ソートが回答数降順 -> 正答率昇順で動作する。
- 検索画面側の共通パーツに回帰がない。
- グラフ要素が表示されない。

## 検証観点
1. タイトルと3行説明の表示確認
2. 初期列構成の確認
3. 初期ソート順の確認
4. ヘッダソートの昇降トグル確認
5. 検索画面の表示・操作回帰確認
6. グラフ非表示確認（DOM含む）
