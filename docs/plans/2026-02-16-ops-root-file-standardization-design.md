# 運用ファイル標準化デザイン（ルート直下 -> ops）

## Goal
- ルート直下の運用ファイルを `ops/` 配下へ段階移行し、プロジェクト本体ファイルと運用資産を分離する。

## 対象
- リポジトリルート直下の運用ファイルのみ。
- 主対象: `.ps1`, `.py`, `.bat`, `.cmd`, `.txt`, `.log`, 運用メモ系 `.md`
- 対象外: `pom.xml`, `README.md`, `AGENTS.md` などプロジェクト本体の基幹ファイル

## 棚卸し仕様
- ルート直下ファイルを一覧化し、各ファイルに次を付与する。
  1. 推奨移動先（`ops/scripts`, `ops/log`, `ops/notes`）
  2. 影響度（High/Medium/Low）
  3. 移動容易性（Easy/Medium/Hard）
  4. 優先度ランク（P1/P2/P3）

## 優先度の決め方（2軸）
- 影響度: 移動ミス時の運用影響。
- 移動容易性: 参照更新の必要量と複雑さ。
- 基本方針: 影響度高 × 容易性高を P1 とし、複雑なものは P2/P3 へ後退させる。

## 実施順序
1. 棚卸し結果を確定する（一覧 + 2軸 + 優先度）。
2. P1（低リスク）から 1コミット1目的で移行する。
3. 参照を持つファイルは、移行と同一コミットで最小限の参照更新を行う。
4. P2/P3 を段階的に実施する。

## 完了条件
- 例外定義を除くルート直下の運用ファイルが `ops/` 配下へ移行済み。
- README/スクリプト内の参照が新パスに一致している。
- 検証コマンド（最低 `mvn compile`）が成功する。

## 運用ルール
- 1コミット1目的を維持する。
- 実行順は「棚卸し -> 移行 -> 検証」を直列で実施する。
- 失敗時は差分を分離し、対象コミット単位でロールバック可能な状態を維持する。
