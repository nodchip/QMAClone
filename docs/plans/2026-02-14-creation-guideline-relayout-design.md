# 問題作成画面ガイドライン再配置・配色方針設計（2026-02-14）

## 目的
- 問題作成画面の情報過多を解消する。
- 「投稿のお願い」「ガイドライン」の導線を分離し、作問フローを中断しにくくする。
- ロビー基準のトーン統一を維持しつつ、用途別コントラスト差を残して可読性を高める。

## 決定事項
- 長文ガイドラインは作成画面から分離し、専用ページへ移設する。
- 作成画面には短文の注意喚起と「ガイドラインを見る（新規タブ）」のみを表示する。
- 配色方針は「ロビー基準（濃紺系）」で統一する。
- ただし用途別にコントラスト差を持たせる。
  - ロビー: 高コントラスト（操作優先）
  - 作成画面: 1段明るい面 + 明るいフォーム面（入力可読性優先）
  - ガイドライン: 明色面 + 暗文字（長文読解優先）

## 情報配置
- `CreationUi` 冒頭に以下を配置:
  - 問題投稿の短文案内（2行）
  - ガイドラインリンク（新規タブ）
- 既存の長文ガイドラインブロックは専用ページへ移設する。
- 専用ページ先頭に要点サマリー（3〜5項目）を置く。

## 文言方針
- 命令調を弱め、行動が明確な依頼文に統一する。
- 見出しは由来説明より実用性を優先する。
  - 例: 「QMAClone 問題作成ガイドライン」
- 由来説明（旧2chスレッド由来）は冒頭注記として短く残す。

## UIトーン設計
- ブランドトーン（濃紺 + シアン系アクセント）は維持する。
- 背景・面・境界の色は共通変数で管理し、画面ごとに明度差のみ調整する。
- ガイドライン本文は明色背景 + 暗文字で統一し、長文可読性を最優先する。

## 検証観点
- 作成画面で主要操作に不要スクロールが発生しない。
- 「ガイドラインを見る」で新規タブ遷移し、作成中入力が保持される。
- 作成画面スクロール時の背景トーン急変が解消される。
- ガイドライン本文の見出し・本文・箇条書きが読みやすいコントラストになっている。
