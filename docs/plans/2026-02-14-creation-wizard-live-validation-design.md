# 問題作成ウィザード 即時バリデーション設計

作成日: 2026-02-14  
対象: QMAClone 問題作成ウィザード

## 1. 目的
- 問題作成時の入力ミスを早い段階で検知し、送信直前の手戻りを減らす。
- ステップごとに必要な入力条件を明確化し、迷わず次に進める導線を提供する。

## 2. 方針
- 即時バリデーションを採用する。
- エラーメッセージは具体的に表示する。
- ステップ上部に未解決エラー件数を表示する。
- `次へ` 押下時にも最終検証を行い、未解決エラーがある場合は遷移をブロックする。

## 3. 責務分離
- `CreationUi`
  - ステップ遷移制御
  - ステップ単位のエラー件数集約表示
  - 遷移時のフォーカス移動制御
- `WidgetProblemForm`
  - 入力値の収集
  - 項目単位のエラー表示（項目下表示）

## 4. バリデーションフロー
1. 入力イベント（Change/KeyUp）で `validateCurrentStepLive()` を実行。
2. `StepValidationResult` に `isValid / fieldErrors / firstErrorFieldId` を格納。
3. `WidgetProblemForm` が項目下エラーを描画。
4. `CreationUi` がエラー件数を更新。
5. エラー0件で `次へ` 活性、1件以上で非活性。

`次へ` 押下時:
- `validateStepForTransition(step)` を実行。
- 失敗時は遷移しない。
- 最初のエラー項目へフォーカスを移動。

## 5. ステップ別仕様
### Step1
- ジャンル未選択: 「ジャンルを選択してください」
- 出題形式未選択: 「出題形式を選択してください」

### Step2
- 問題文空: 「問題文を入力してください」
- 空白のみ: 「問題文は空白のみでは登録できません」

### Step3
- 必須解答欠落: 「解答1を入力してください」
- 形式整合: 既存 `validateProblem()` / type validator の文言を再利用
- 外部コンテンツURL不正: 「URLの形式が正しくありません」

## 6. UI表示ルール
- エラー項目ラベルを赤色化。
- 項目下に1行エラーメッセージを表示。
- 修正後は即時でエラーを消去。
- ステップ上部に「n件の入力エラーがあります」を表示。

## 7. テスト観点
1. Step1で未選択のまま `次へ` を押すと遷移しない。
2. Step2で空問題文のとき即時エラー表示される。
3. Step2で入力すると即時でエラーが消える。
4. Step3で必須解答欠落時に遷移しない。
5. Step3整合OKでStep4へ遷移できる。
6. Step4から戻って修正後、再検証に反映される。

回帰確認:
- 送信確認画面遷移と問題送信が維持される。
- 問題コピー/問題修正読み込み後も同仕様で動作する。
