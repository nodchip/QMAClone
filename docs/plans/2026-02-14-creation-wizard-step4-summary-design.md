# 問題作成ウィザード 確認ステップ情報整理設計

作成日: 2026-02-14  
対象: QMAClone 問題作成ウィザード Step4

## 1. 目的
- 確認ステップでの読み取り負荷を下げる。
- 送信前確認を短時間で行えるようにする。
- 修正が必要な場合の戻り導線を明確にする。

## 2. 方針
- 表示方式は「要点サマリー + 詳細を開く」を採用する。
- サマリーはセクション単位（基本情報 / 問題文 / 解答設定）で提示する。
- 詳細はセクション単位で独立トグルする。

## 3. セクション仕様
### 3.1 基本情報
- サマリー: ジャンル / 出題形式 / ランダムフラグ / 作成者
- 操作: 「編集へ戻る」ボタンで Step1 に戻る

### 3.2 問題文
- サマリー: 先頭40文字 + 文字数
- 補助: 空または短すぎる場合は警告表示
- 操作: 「編集へ戻る」ボタンで Step2 に戻る

### 3.3 解答設定
- サマリー: 解答1、選択肢数、外部コンテンツ有無
- 補助: 画像回答・画像選択肢はタグ表示
- 操作: 「編集へ戻る」ボタンで Step3 に戻る

## 4. 挙動仕様
- 初期表示は3セクションすべて折りたたみ（サマリーのみ表示）。
- 「詳細を開く」で対象セクションのみ展開する。
- 他セクションは開閉状態を維持する（相互排他にしない）。
- 展開内容は表示専用とし、編集は各ステップへ戻って行う。

## 5. エラー連動
- エラーがあるセクションは赤枠で強調する。
- セクションタイトルにエラー件数を表示する。
- `送信確認画面に移動` は全セクション正常時のみ活性にする。

## 6. 実装方針
- `CreationUi.ui.xml` の Step4 を3つの summary card 構造に変更する。
- `CreationUi.java` にセクションデータ構築処理を追加する。
- 既存 `updateStep4Summary()` は段階的に置換する。
- トグル状態は画面内メモリで保持し、永続化しない。

## 7. テスト観点
1. Step4到達時に3セクションが表示される。
2. 詳細トグルが対象セクションのみ開閉する。
3. 「編集へ戻る」で対応ステップに遷移する。
4. エラー時に該当セクションが警告表示される。
5. 問題文サマリーが40文字で省略表示される。

## 8. 検証コマンド
- `mvn -q compile`
- `mvn -q test`
- `mvn -q "-Dgwt.skipCompilation=false" gwt:compile`
