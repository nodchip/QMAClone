# ロビーCTA下 設定UIモダン化デザイン

## Goal
- ロビー画面の CTA（対戦開始ボタン群）下にある設定項目の視線誘導を改善し、優先設定と詳細設定の階層を明確にする。

## 方針
- 2段構成で情報階層を作る。
  - 上段（主要設定）: `ジャンル / 出題形式 / 難易度 / プレイヤー名`
  - 下段（詳細設定）: `地域 / 挨拶 / 新問設定 / イベント名・公開 / テーマ`
- ミニマル刷新とし、既存の色トーンは維持する。
- ロジック影響を抑えるため、`ui:field` とイベント配線は維持し、主に UiBinder 構造とCSSで解決する。

## 情報設計
- CTA直下に「主要設定」セクションを配置し、最初に決めるべき項目を集約する。
- 「詳細設定」は1カードにまとめ、補助項目として二次的に閲覧できる構成にする。
- 現在のフラットな1テーブル構成を、優先度が伝わる構造へ整理する。

## UIスタイル
- セクション見出しを追加（`主要設定` / `詳細設定`）。
- 上段カードは余白と入力欄密度をやや高め、操作起点にする。
- 下段カードは現行密度に近いまま維持し、補助情報の視認性を担保する。
- `ListBox` / `TextBox` のフォーカス・境界・背景を統一し、入力体験の揺れを抑える。

## 実装範囲
- 主対象:
  - `src/main/java/tv/dyndns/kishibe/qmaclone/client/lobby/LobbyUi.ui.xml`
- 必要に応じて:
  - `src/main/webapp/QMAClone.css`（既存ロビー変数と整合する最小追加）

## 完了条件
- CTA下の設定が2段構成（主要/詳細）で表示される。
- 既存操作（イベント参照ボタン、公開チェック、各入力）に回帰がない。
- `mvn compile` が成功する。
- デプロイ後に `/QMAClone-1.0-SNAPSHOT/` が `HTTP 200`、`/tv.dyndns.kishibe.qmaclone.QMAClone/service` が `HTTP 405` を返す。
