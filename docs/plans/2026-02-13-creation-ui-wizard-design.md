# 問題作成UI ウィザード化設計

作成日: 2026-02-13  
対象: QMAClone 問題作成画面

## 1. 合意事項
- 対象画面: 問題作成
- 目標: 入力効率 / ミス防止 / 学習しやすさ を同時に高水準化
- 方式: ステップ分割
- 導線: 上部ステッパー
- 自動保存: ステップ遷移時に実行

## 2. 画面構造
問題作成を4ステップに分割する。

1. 基本情報
2. 問題文
3. 解答設定
4. 確認

### 2.1 レイアウト
- 上部: ステッパー（現在地/完了/未着手）
- 中央: ステップ本体カード
- 下部: 共通操作（戻る / 次へ / 保存 / 公開）

### 2.2 遷移ルール
- 後戻りは常時可能
- 先ステップへのジャンプはバリデーション通過後のみ
- URL遷移ではなく同一画面内切替で実装

## 3. バリデーション方針
ステップごとに検証責務を分割する。

- Step1: 基本メタ情報の必須チェック
- Step2: 問題文の必須/文字数/形式チェック
- Step3: 解答・選択肢・形式依存制約の整合
- Step4: 全体整合チェック

### 3.1 実行順序
- `次へ` 押下時: `validate -> autoSave -> move`
- どこかで失敗した場合は遷移しない

### 3.2 エラー表示
- 項目直下にエラー
- 上部に要約バナー
- 確認ステップではエラー項目から該当ステップへ戻せる導線を用意

## 4. 自動保存方針
- トリガ: ステップ遷移時
- 差分保存: 変更がある場合のみ保存API呼び出し
- 成功時: トースト表示
- 失敗時: 上部エラーバナー + 再試行ボタン

## 5. 実装方針（段階導入）
### Phase 1: UI骨格
- ステッパーと4ステップ枠を導入
- 既存入力ウィジェットを移設（ロジックは温存）

### Phase 2: ステップ検証
- `validateStep(stepId)` を導入
- 遷移阻止ルールを適用

### Phase 3: 遷移時自動保存
- dirty判定による差分保存
- 保存失敗時の再試行導線を追加

### Phase 4: 確認ステップ強化
- サマリー表示
- 差し戻しリンク

## 6. ファイル責務
- UI構造: `CreationUi.ui.xml`（または新規ウィザードUI xml）
- 遷移制御: `CreationUi.java`
- 保存/検証実処理: 既存 Presenter/Service に委譲
- 追加モデル: `StepState`（現在ステップ、完了状態、dirty）

## 7. 検証計画
毎フェーズで直列実行する。

1. `mvn compile`
2. `mvn test`
3. `mvn "-Dgwt.skipCompilation=false" gwt:compile`
4. デプロイ

### UI確認観点
- 正常遷移（保存成功）
- 検証失敗時の遷移阻止
- 保存失敗時の再試行導線

## 8. 完了条件
- 4ステップで問題作成を完了できる
- 途中離脱から再開できる
- 入力ミス時の修正導線が明確
