# 2026-02-18 Result Screen Modernization Design

## 目的
- 結果表示画面の情報伝達速度を上げ、対戦直後に「自分の結果」と「全体順位」が1画面で直感的に把握できるUIにする。
- 既存のゲーム本体デザイン（青系トーン、角丸、カード表現）との統一感を維持する。
- 既存機能（結果取得、ロビー遷移、問題レポート表示）を壊さず、見た目と導線のみを改善する。

## 現状整理
- 現在は `PanelResult` が `Grid` を使って一律表示しており、情報の重要度が視覚上で分離されていない。
- 「自分の結果」「全体順位」「遷移リンク」「問題レポート」が同じ密度で並び、試合直後に注目すべき情報へ視線誘導しづらい。
- レーティング変化は矢印文字列で表現されるが、比較のしやすさと視認性に改善余地がある。

## 採用方針（情報優先カード型）
### 1. 画面構成
- `PanelResult` を3層構成に再編する。
1. `resultHero`: 自分の結果サマリー（順位・得点・レート変動）
2. `resultRankingList`: 全員の順位カード一覧
3. `resultFooter`: 「ロビーに戻る」導線と問題レポート
- 重要情報を上、詳細を中、補助情報を下に配置し、情報の優先順位を明確化する。

### 2. 表示ルール
- ヒーロー領域は `SceneResult` で特定した `myResult` のみを表示する。
- ランキング領域は全プレイヤーを同一カードフォーマットで表示する。
- 自分の行のみ背景トーンを変えて強調する。
- レート変動は `+N / -N / ±0` バッジ形式で表示し、色は上昇=緑、下降=赤、維持=ニュートラルで統一する。
- `newRating <= 0` は「変動なし」表示にフォールバックする。

### 3. 状態別UI
- `loading`: スケルトンまたはプレースホルダー表示。
- `ready`: 結果カード表示。
- `error-light`: 再試行中を小さく通知。
- `error-heavy`: 中央エラー表示＋ロビー遷移導線を明示。
- `RetryAsyncCallback` の既存再試行方針は維持し、見せ方だけ整える。

## 実装計画
### ステップ1: 構造分離
- 対象: `src/main/java/tv/dyndns/kishibe/qmaclone/client/PanelResult.java`
- `Grid` 中心構造を `FlowPanel` / `VerticalPanel` ベースへ移行。
- コンテナクラスを明示（`resultHero`, `resultRankingList`, `resultFooter`）し、CSS責務を明確化。

### ステップ2: スタイル適用
- 対象: `src/main/webapp/QMAClone.css`
- 結果画面専用クラスを追加し、既存の色変数・角丸・影を再利用。
- チャット展開時でも横はみ出ししない幅制御（`box-sizing`, `max-width`）を入れる。

### ステップ3: 微アニメーションと例外表示
- ヒーロー領域のみ短いフェードイン。
- ランキングカードは軽いスタッガー（過剰な演出はしない）。
- 失敗時メッセージの視認性を改善。

## 検証計画
### 機能検証
1. 結果取得成功時に自分サマリーと全員カードが表示される。
2. レート表示が上昇・維持・下降で正しく分岐する。
3. `newRating <= 0` で表示崩れせず「変動なし」になる。
4. 軽微失敗時に再試行導線が機能する。
5. 重大失敗時にロビー遷移導線が機能する。

### レイアウト検証
1. 通常デスクトップ幅で崩れない。
2. 横幅を狭めた状態でもカードがはみ出さない。
3. チャット展開時に中央パネルと重ならない。

## 完了条件
1. 自分の結果が最上段で即時判読できる。
2. 全順位が同一フォーマットで比較しやすい。
3. 問題レポート導線を維持する。
4. ゲーム全体のデザイン言語と統一される。
