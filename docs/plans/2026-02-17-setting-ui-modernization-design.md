# 設計画面 UI/UX モダン化設計（2026-02-17）

## 1. 背景と目的
- 対象: 設定画面の全タブ（`その他` / `ユーザーコード` / `チャット` / `テーマモード` / `管理者用`）。
- 現状課題:
1. タブごとに文言トーンや操作配置がばらついている。
2. 保存は即時反映されるが、失敗時のユーザー向けフィードバックが弱い。
3. 説明テキストと操作UIの情報階層が一定でなく、初見時に迷いやすい。
- 目的:
1. 操作の分かりやすさを優先して全タブを横断的に統一する。
2. 保存成功/失敗をユーザーが即座に認識できる状態にする。
3. 見た目と構造を共通ルール化し、今後の拡張時にも崩れにくい形にする。

## 2. 仕様確定（合意事項）
### 2.1 保存と通知
- 保存方式は即時反映を維持する。
- 成功通知は右下トースト（既存方針）を継続する。
- 表示は項目名入り（例: `「効果音」を保存しました`）。
- 同時表示は1件のみ（新しい通知で上書き）。
- 表示時間は2.0秒。
- 配色は既存テーマ準拠の青系。
- 失敗時は失敗トーストを新設し、同位置で表示する。

### 2.2 操作導線と文言
- 文言トーンは行動ベースに統一する（保存/切り替え/解除/更新）。
- ボタン配置は次で統一する。
1. 主要操作: 右寄せ
2. 補助操作: 左寄せ

### 2.3 レイアウト構造
- 各タブを次の2層構造に統一する。
1. 説明文カード（何ができるか、注意点）
2. 操作カード（入力・選択・実行）
- 余白、ラベルと入力の整列、カード見出しスタイルを全タブで共通化する。

## 3. 実装方針
### 3.1 共通化方針
- 大規模なロジック置換は避け、既存コンポーネントを活かしながら以下を中心に実装する。
1. CSS クラスの再編と共通クラス追加
2. 文言の短文化・統一
3. 操作領域の再配置
4. 保存成功/失敗トーストの接続

### 3.2 タブ別の重点
- `その他`: 旧来の項目群を説明/操作の2カードへ再配置し、行構成を統一。
- `ユーザーコード`: 切替・連携・解除を目的単位で分離し、文言を行動ベースへ統一。
- `チャット`: ON/OFF 設定と補助操作の境界を明確化。
- `テーマモード`: 編集操作と権限申請を区別し、誤操作を減らす。
- `管理者用`: 破壊的操作（クリア/却下）を補助操作扱いで視覚的に区別。

## 4. エラーハンドリング方針
- 成功と失敗を同じ通知コンポーネント系統で扱い、表示位置を統一する。
- 失敗時は処理単位で項目名を含む文言を表示する。
- 内部実装語（クラス名、コールバック名）はユーザー向け文言に出さない。

## 5. 検証観点（完了条件）
- 各タブで即時保存時に成功トーストが正しく表示される。
- 保存失敗を発生させた際に失敗トーストが表示される。
- 主要操作右寄せ/補助操作左寄せが全タブで統一される。
- 説明文カード + 操作カードの2層構造が全タブで揃う。
- build -> test -> deploy を直列実行し、デプロイ疎通確認を満たす。

## 6. 実施順（優先）
1. 操作の分かりやすさ（文言・導線・配置）
2. 安心感（成功/失敗通知）
3. 見た目統一（カード/余白/タイポ）
