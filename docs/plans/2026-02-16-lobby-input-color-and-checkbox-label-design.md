# ロビー設定 入力配色・チェックラベル改行デザイン

## Goal
- ロビー画面 CTA 下の設定項目において、入力部品の配色を統一し、チェックボックスラベルの不自然な改行を解消する。

## 方針
- 1回目は最小変更で実施する（`LobbyUi.ui.xml` のローカルCSSのみ）。
- 2回目で必要に応じて `QMAClone.css` に共通化して昇格する。

## 対象
- コンボボックス（`ListBox`）
- テキストボックス（`TextBox`）
- チェックボックス本体
- チェックボックスラベル

## 配色方針（最小変更）
- ダークカードに合わせた「濃紺ベース + 淡い文字」で統一する。
- 入力欄背景は濃紺、文字は淡色、境界線は青系に統一する。
- フォーカス時は既存アクセント系色で視認性を上げる。

## 改行方針
- チェックボックスラベルは1行固定とする。
- `white-space: nowrap;` を適用し、意図しない折り返しを抑止する。

## 実装範囲（第1段）
- `src/main/java/tv/dyndns/kishibe/qmaclone/client/lobby/LobbyUi.ui.xml`
  - ローカルCSS追加/調整のみ
  - `ui:field` やロジックには手を入れない

## 次段（第2段）
- ローカルCSSで安定確認後、`QMAClone.css` へ共通クラス化して昇格する。
- 共通化時はトークン化と意味付きクラス名を付与し、他画面へ段階展開可能にする。

## 完了条件
- 主要/詳細カードの入力部品配色が統一される。
- チェックボックスラベルが1行固定で表示される。
- `mvn compile` が成功する。
- デプロイ後、`/QMAClone-1.0-SNAPSHOT/` が `HTTP 200`、`/tv.dyndns.kishibe.qmaclone.QMAClone/service` が `HTTP 405` を返す。
