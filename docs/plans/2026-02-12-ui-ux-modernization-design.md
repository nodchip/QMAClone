# QMAClone UI/UX モダン化 設計

## 1. 目的
- QMAClone 全体の UI/UX をモダン化し、可読性・視認性・体感性能を改善する。
- 既存機能を壊さず、全画面に共通する見た目と操作体系を統一する。

## 2. 成果目標
- 可読性最優先: 本文 17-18px を基準に、コントラストと階層を明確化する。
- ナビゲーション改善: 大分類と小分類を分離し、迷いにくい導線にする。
- チャット改善: 常時フル表示はせず、右下ドックと独立画面を併用する。
- 体感性能改善: 画面遷移・開閉・入力の反応を損なわない。

## 3. デザイン方針
- トーン: eスポーツ風（VALORANT Esports + FACEIT の中間）。
- 配色: ネイビー基調 + オレンジ/ティール。
- モーション: 中程度（過剰演出を避ける）。
- 画面構造: サイドバー（大分類）+ 画面内タブ（小分類）+ チャットドック。

## 4. 情報設計
### 4.1 大分類（サイドバー）
- ホーム
- 対戦
- 問題作成
- ランキング
- コミュニティ
- 設定

### 4.2 小分類（画面内タブ）
- 各大分類内で 4-6 項目を上限に設定する。
- 「画面単位」ではなく「操作目的単位」で分割する。

### 4.3 チャット
- 右下ドック（開閉）+ 未読バッジ + 独立画面を併用する。
- ロビーでは初期展開、対戦中は折りたたみ既定にする。
- 通知は入力フォーカスを奪わない。

## 5. アーキテクチャ
### 5.1 スタイル基盤
- `src/main/webapp/QMAClone.css` を中心にデザイントークンを導入する。
- 色・余白・角丸・影・タイポグラフィ・モーション時間を `:root` 変数化する。
- 既存 `.gwt-Button-*` / `.grid*` などは互換レイヤで維持し、段階移行する。

### 5.2 レイアウト基盤
- `src/main/webapp/QMAClone.html` に共通コンテナを追加する。
  - サイドバー領域
  - 上部タブ領域
  - メイン表示領域
  - チャットドック領域
- 既存 GWT マウントポイントとの競合を避け、互換モードを維持する。

### 5.3 UI 実装方針
- 今回変更・追加する表示のみ UiBinder を利用する。
- 既存の未変更領域は保留し、全面 UiBinder 化は行わない。

## 6. データフロー / 状態管理
- 画面共通 UI 状態を軽量に一元管理する。
  - 現在の大分類
  - 現在の小分類タブ
  - チャット開閉状態
  - チャット未読件数
- 各画面固有の業務状態は既存ロジックを維持し、UI状態と分離する。

## 7. エラーハンドリング
- チャットドック初期化失敗時は独立チャット画面へ自動フォールバックする。
- サイドバー/タブ描画失敗時は既存画面構成へフォールバックする。
- CSS 読み込み失敗時は既存スタイルで継続利用可能にする。

## 8. 実行順序（骨格先行）
1. 共通デザイントークンと互換レイヤを `QMAClone.css` に追加する。
2. 共通レイアウト骨格を `QMAClone.html` に追加する。
3. ナビゲーション状態管理とチャットドック制御を実装する。
4. 主要画面を段階適用する（ロビー/対戦導線/設定/ランキング）。
5. 高リスク領域（ゲーム入力パネル群）は見た目適用を先行し、構造変更は後段にする。

## 9. 検証方針
- 実装後は `build -> test` を直列実行する。
- 手動シナリオを固定化して確認する。
  - ロビー -> 対戦 -> 結果 -> 設定
  - チャットドック開閉 / 未読反映 / 独立画面遷移
- レスポンシブ確認は Desktop/Mobile の 2 系統で実施する。
- 可読性確認として、フォーカス可視化・文字サイズ・コントラストを確認する。

## 10. 非目標
- 今回は全面 UiBinder 化を行わない。
- 今回は業務ロジックの再設計を行わない。
