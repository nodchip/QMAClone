# 2026-02-16 チャットUIモダン化設計

## 背景
- 全般的なUI/UXモダン化の継続として、チャット領域を対象にする。
- 直近でロビー画面は改善済みのため、次はチャットの視認性と見た目品質を引き上げる。

## 目的
- 視線移動を減らし、チャットの状態と操作可否を一目で分かるようにする。
- 現行機能（送受信、折りたたみ、状態復元）を維持したまま、見た目をモダン化する。

## スコープ
- 対象:
  - チャット表示本体（PanelChat 相当）
  - 折りたたみ表示制御（Controller 相当）
  - チャット関連CSS
- 非対象:
  - RPC/データ構造
  - ゲーム本体の機能ロジック
  - 他画面の全面リデザイン

## 決定事項
- 実装方針: 一気に刷新
- デザインテイスト: フラット上質（装飾は控えめ、境界と階層で見せる）
- 制約: DOM/情報構造の制約を設けず最適化可能
- 表示位置: 右下固定フローティング

## UX設計
### 情報階層
- 3層構造に統一する。
  1. ヘッダー（タイトル、状態、主要操作）
  2. メッセージ領域（本文中心、通知との区別）
  3. 入力領域（入力欄と送信導線）

### 配置
- デスクトップ基準:
  - `right: 20px; bottom: 20px`
  - 展開時 `width: 360px; height: min(56vh, 520px)`
- 折りたたみ時:
  - ピル状トリガを常時表示
  - `height: 40px` 前後、`min-width: 160px`
- 狭幅時:
  - `right: 12px; bottom: 12px; width: calc(100vw - 24px); max-width: 420px`
- 開閉時は同一アンカーで位置ジャンプを禁止する。

### ビジュアルトークン
- 外枠: `#0c223d`
- ヘッダー: `#123052`
- メッセージ領域: `#0a1d34`
- 入力領域: `#0f2744`
- 境界線: `#3f6f99`（1px）
- 文字:
  - 本文 `#e6f2ff`
  - 補助 `#b8cfe3`
  - 無効 `#7893aa`
- 角丸:
  - 外枠 12px
  - 内部 8px
- 影: `0 10px 24px rgba(0,0,0,.28)`

### 操作フィードバック
- 送信ボタン: 通常/hover/active/disabled の視覚差を必須化する。
- 入力フォーカス: `outline: 2px solid #69deff`。
- 展開/折りたたみアニメーション: `160ms ease-out`。
- メッセージ追加アニメーションは導入しない（可読性優先）。

## 実装方針
1. 右下固定レイアウトを導入する。
2. `chat-*` プレフィックスのCSS変数を定義する。
3. ヘッダー/本文/入力の3層カード構造へ整理する。
4. 折りたたみ時トリガの常時視認と再展開導線を保証する。

## 影響範囲
- 変更はチャット関連コンポーネントとチャット関連スタイルに限定する。
- 他画面のUIには副作用が出ないよう、既存クラスを汚染しない。

## 検証計画
- 手順: `mvn compile` -> `mvn test` -> `deploy_qmaclone_tomcat9.ps1`
- 配備判定:
  - `/QMAClone-1.0-SNAPSHOT/` が HTTP 200
  - `/tv.dyndns.kishibe.qmaclone.QMAClone/service` が HTTP 405
- 画面確認:
  1. 通常表示（右下固定、主要UI非干渉）
  2. 折りたたみ表示（常時トリガ視認）
  3. 折りたたみ解除（位置ジャンプなし）
  4. 狭幅表示（はみ出しなし）
  5. 送信ボタン状態差（通常/hover/active/disabled）

## 完了条件
- 右下固定チャットが意図した配色・階層・操作フィードバックで表示される。
- 折りたたみ導線が常時有効で、再展開が1アクションで可能。
- 既存チャット機能（送受信・状態復元）に回帰がない。
