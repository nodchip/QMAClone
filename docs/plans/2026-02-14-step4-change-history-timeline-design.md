# Step4 編集履歴 UI 設計（2026-02-14）

## 目的
- Step4 の編集履歴を、作業中でも読み取りやすい表示へ改善する。
- 「最新の変更把握」と「2履歴比較による差分確認」を両立する。

## 採用方針
- 表示形式は「タイムライン型カード」にする。
- 並び順は「新しい順（最新が上）」にする。
- 初期展開は「最新1件のみ」。2件目以降は折りたたみ。
- 各履歴カードに以下を配置する。
  - 日時
  - プレイヤー
  - 変更サマリープレビュー
  - 変更前 / 変更後 選択ラジオ
  - 詳細開閉ボタン
- 差分プレビュー領域はカード一覧の下に固定表示する。

## データ・挙動
- 既存 API (`getProblemCreationLog`, `generateDiffHtml`) は変更しない。
- 履歴データ受信後、クライアントで日付降順に並べ替える。
- 「変更前」「変更後」ラジオ選択時は従来どおり Presenter 経由で差分生成する。
- サマリー未設定の履歴は「変更サマリー: なし」と表示し、詳細開閉は無効化する。

## 非対象
- 差分生成ロジック自体の変更
- 履歴取得 API の拡張
- Step4 以外の画面遷移仕様変更

## 検証
1. Step4 で編集履歴が新しい順で表示される。
2. 初期表示で最新1件のみ詳細表示される。
3. 2件目以降は「詳細を開く」で展開できる。
4. 変更前/変更後を選択すると差分プレビューが更新される。
