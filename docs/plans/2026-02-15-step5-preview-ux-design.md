# Step5「問題出題プレビュー」UI/UX 設計

## 背景
- 対象は問題作成ウィザード Step5（確認）にある「問題出題プレビュー」。
- 現在は問題出題画面と共通パーツを利用しており、共通化を維持したい。
- 目的は「見本としての分かりやすさ」を上げることであり、共通パーツ自体の改変は最小化する。

## 決定事項（確定）
1. 改修方針は `最小追加`。
- 共通パーツ本体は変更しない。
- Step5 側に補助文のみ追加する。

2. 補助文の内容は `最小`。
- 文言は1行のみとする。
- 文言: 「以下は、実際の出題画面に近い表示です。」

3. 補助文の配置は `プレビュー直上`。
- 見本直前で意味付けを行う。

4. 表示条件は `プレビュー表示時のみ`。
- Step5 遷移直後には表示しない。
- プレビューが表示されたときだけ表示する。
- プレビューが非表示に戻る場合は補助文も非表示に戻す。

## 情報階層
- Step5 の主要導線（確認ボタン群）
- 補助文（プレビュー表示時のみ）
- 問題出題プレビュー（共通パーツ）

これにより、説明文の先出しを避け、視線の流れを「操作 -> 説明 -> 見本」に揃える。

## 実装方針
- 変更対象は Step5 のラッパー層（`CreationUi` / `CreationUi.ui.xml` / 必要最小限の CSS）。
- 共通の問題出題画面パーツには変更を入れない。
- 既存の Step5 表示制御フラグに補助文を連動させる。

## 状態遷移
1. Step5 初期表示
- プレビュー非表示
- 補助文非表示

2. プレビュー表示トリガ発火後
- プレビュー表示
- 補助文表示

3. プレビュー非表示化
- プレビュー非表示
- 補助文非表示

## エラーハンドリング方針
- 補助文は RPC など非同期レスポンスに依存させない。
- プレビュー表示/非表示の状態のみで制御する。
- 表示更新経路が複数ある場合は、表示制御メソッドを共通化して片寄りを防ぐ。

## 検証観点
1. Step5 遷移直後は補助文が表示されない。
2. プレビュー表示時のみ補助文が表示される。
3. プレビュー非表示時に補助文も非表示へ戻る。
4. 問題出題画面（共通パーツ利用側）へ見た目回帰がない。
5. 類似問題セクションの表示制御と干渉しない。

## 実装スコープ外
- 問題出題画面の共通パーツデザイン変更。
- Step5 以外の文言・レイアウト刷新。
