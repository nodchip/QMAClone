# war 非推奨化ゲート最終化デザイン

## Goal
- `war/` 非推奨化の残タスクを完了し、フォルダ標準化の次フェーズへ進める状態にする。

## Scope
- `deploy_qmaclone_tomcat9.ps1` を使った通常配備の実機確認。
- HTTP ステータス確認（200/405）と再読み込み時 404 非発生の確認。
- 実行証跡を計画書へ追記し、失敗時の切り分け順序を固定化。

## 実行手順
1. 事前ビルド確認
- `mvn compile` を実行し成功を確認する。

2. 通常配備
- `deploy_qmaclone_tomcat9.ps1` を既定手順で実行する。
- 回避的なスキップオプションは使わない。

3. HTTP 検証
- `/QMAClone-1.0-SNAPSHOT/` が HTTP 200。
- `/tv.dyndns.kishibe.qmaclone.QMAClone/service` が HTTP 405。

4. 再読み込み安定性検証
- 配備直後に再読み込みを複数回実施し、HTTP 404 が発生しないことを確認する。

5. 証跡記録
- `docs/plans/2026-02-15-war-directory-deprecation-plan.md` に以下を追記する。
  - 実行時刻
  - 実行コマンド
  - 200/405 の結果
  - 再読み込み試行回数と 404 発生有無

## 完了条件
- 通常配備が成功している。
- 200/405 を取得できる。
- 再読み込みで 404 が再現しない。

## 失敗時の扱い
- 推測で断定せず、以下の順で切り分ける。
1. 接続可否（Tomcat 到達）
2. HTTP ステータス（200/405/404）
3. アプリ例外（Tomcat ログ）

- 復旧が必要な場合は Git ベースで戻す。手動コピーなどの ad-hoc 手順は採用しない。

## 次フェーズ接続
- 本ゲート完了後、ハイブリッド方針に従って次を開始する。
1. Maven 標準寄せ（構造）
2. `ops/` 段階移行（運用）
